- pipeline: whether the workflow contains the expected scientific stages

- implementation: whether computations and code logic are correct
- analysis: whether the analysis extracts a meaningful scientific result
- validation: whether the result is checked, stress-tested, or compared against alternatives

Important benchmark design constraints
================================

2. L1/L2/L3 should differ primarily in autonomy, not in physics target.
3. Do not simply copy the reference analysis verbatim into all prompts.
4. L1 should be explicit but not bloated.

10. The private rubrics should reflect the reference analysis but should not assume the reference is the only valid solution.

    - "uses physically motivated photon selection"
    over unnecessarily narrow checks like:
    - "must use exactly X library"
    - "must use exactly one specific plotting style"
13. Make each private rubric practically useful for later deterministic checks, heuristics, or LLM-judge evaluation.
